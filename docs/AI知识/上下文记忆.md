**核心机制**
- 短期上下文窗口
  - 把当前对话若干轮消息直接拼进模型上下文，保证最近交流不丢失
  - 使用“滚动摘要”把更早的消息压缩成要点，释放 token 空间
- 长期结构化记忆
  - 把稳定事实写入“核心记忆”：如项目技术栈、约定、你的偏好与约束
  - 典型分层：用户级（通用偏好）、项目级（该仓库的约定与术语）
- 语义检索（RAG）
  - 对历史对话、文档、代码切块做向量化；按当前问题语义 Top‑K 召回
  - 结合多策略排序：余弦相似度 + 新近度 + 多样化（MMR）去重
- 代码索引
  - 独立维护代码库索引（符号/路径/AST/依赖关系），随文件变更实时更新
  - 从“打开文件、相关定义、被引用处”优先取上下文，比单纯全文检索更稳

**记忆粒度**
- 会话级
  - 记录你的明确偏好与规则（如“不重启项目、不跑 check”）
  - 任务讨论的阶段性结论会被压成摘要，供后续继续执行
- 项目级
  - 技术框架、文件约定、请求封装、路由/状态管理模式等稳定事实
  - 领域术语映射与命名约定，避免术语漂移
- 代码级
  - 活动文件与附近函数/类型定义
  - 与当前问题最相关的实现片段与调用关系

**检索与拼装（具体到步骤）**
- 收集候选
  - 语义检索历史对话/文档/issue/wiki/代码块 Top‑K
  - 代码索引按符号与文件邻域补充高优先级候选
- 排序与筛选
  - 评分 = 语义相似度×权重 + 新近度×权重 + 使用频次（点击/编辑）×权重
  - MMR 去冗余，保证信息覆盖而不重复
- 上下文预算
  - 预留生成空间；给 system/约束/计划分配固定 token 比例
  - 再按得分从高到低加入候选，超过阈值即截断
- 最终提示词结构
  - 系统约束（风格、权限、不得做的事）
  - 会话摘要（历史要点）
  - 稳定事实（项目/用户偏好）
  - 任务现状与明确目标
  - 检索到的代码/文档片段（带来源）
  - 明确输出要求与边界

**遗忘与更新**
- 压缩与滚动摘要
  - 超过阈值的历史对话被周期性压缩成“要点列表”
- 过期与冲突处理
  - 新事实覆盖旧事实（如升级了路由方案），旧的标记为过期或删除
- 噪声控制
  - 只把“稳定、可复用”的内容写入长期记忆；一次性操作留在短期上下文
  - 代码片段不长期存，只保存指针（文件路径+行号/符号），实时读取最新文件

**隐私与控制**
- 存储范围
  - 本地优先（项目索引/嵌入），必要时云端向量服务；按用户/项目隔离
- 可控开关
  - 允许手动清理对话记忆、关闭项目级记忆、屏蔽敏感文件目录
- 安全
  - 重要内容加密存储；对密钥/凭据做红线过滤，不写入记忆与提示词

**常见问题与应对**
- 上下文漂移
  - 在提示词前置“项目约定与约束块”，强制对齐风格与禁令
- 记忆污染
  - 新增/修改“核心记忆”需要显式动作；合并时包含冲突检测
- 多仓混淆
  - 以工作区路径+Git remote 作为命名空间，跨仓库不共享项目级记忆
- 大代码库遗漏
  - 多通道检索（符号/路径/语义），并在 UI 中暴露“被使用的上下文来源”，方便人工补充/剔除

**对使用者的实操建议**
- 把稳定规则写成项目记忆（例如 docs/ 规则、依赖策略、发布流程）
- PRD/规范用明确前缀与标题，便于结构化提取（如“功能：”“验收：”）
- 讨论中把阶段结论/限制项显式列点，便于滚动摘要保留关键
- 在重要任务开始时，要求助手“记录为项目记忆”，并复述核对
- 多仓并行时，确保在正确工作区里与助手对话，减少命名空间串扰
