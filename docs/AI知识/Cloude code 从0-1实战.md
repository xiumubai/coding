# Claude Code 从0到1实战全攻略知识笔记
本内容为Claude Code（视频中笔误为cloud code）的完整实战教程，核心是将这款AI编程Agent从入门工具转化为生产力利器，同时掌握同类编程Agent的通用使用逻辑，教程分为**环境搭建与基础交互、复杂任务处理与终端控制、多模态与上下文管理、高级功能扩展与定制**四大核心模块，以下是核心知识点总结：

## 一、环境搭建与基础交互
1. **安装与启动**：从官方网站复制命令在终端执行完成安装，通过`mkdir 创建目录+cd 进入目录+cloud 命令`启动工具。
2. **登录与授权**
   - 主动触发登录：`-login`命令，支持**订阅制（Claude Pro/Max会员）**和**API Key按token计费**两种官方方式；
   - 兼容国产模型：可通过设置环境变量，用GLM、Minimax等国产模型驱动，不与Claude模型绑定。
3. **核心：三种操作模式（shift+tab循环切换）**
   - **默认模式**：标注`问号for shortcuts`，创建/修改文件前必询问用户，最稳妥；
   - **自动模式**：标注`accept edit on`，本次会话内自动执行所有文件操作，无需确认，最便捷；
   - **规划模式**：标注`plan mode on`，仅讨论方案、确定细节，不执行任何文件操作，适合复杂需求构思。
4. **基础操作技巧**：终端输入框换行用`shift+回车`；用现代化编辑器替代终端输入框，按`ctrl+g`打开VS Code标签页编辑需求。

## 二、复杂任务处理与终端控制
1. **执行终端命令**：输入`!`进入Bash模式，可运行任意终端命令（如`open 文件名`打开文件）。
2. **规划模式实战**：复杂需求（如架构重构）先进入规划模式确定方案，工具会生成包含目标、目录结构的详细计划，支持**执行+自动模式、执行+默认模式、继续修改计划**三种选择。
3. **权限跳过（双刃剑）**
   - 命令：启动时加`dangerously-skip-permissions`，进入`bypass permissions`模式；
   - 效果：跳过所有终端命令权限检测，自动执行创建目录、安装依赖、删除文件等操作；
   - 风险：获得与用户相同的终端权限，存在理论上的系统风险，需谨慎使用。
4. **后台任务管理**
   - 服务阻塞解决：`ctrl+b`将运行中的服务（如`npm run dev`）放入后台，不影响工具接收新请求；
   - 任务查看：`/tasks`命令查看后台任务，按`k`关闭指定服务，按`esc`返回原界面。

## 三、多模态与上下文管理
### （一）版本回滚（Rewind）
1. 触发方式：`go reverse`命令，或按**两下esc**进入回滚页面，工具会为每次请求创建回滚点；
2. 回滚选项：可选择**回滚代码+对话、仅回滚对话、仅回滚代码、放弃回滚**；
3. 局限性：仅能回滚工具自身写入的文件，终端命令（`mkdir`/`npm install`）生成的文件无法回滚，精准回滚建议用Git；
4. 完整回滚：先关闭相关后台任务，回滚后手动删除终端生成的冗余文件。

### （二）多模态处理：图片与设计稿还原
1. **图片处理**：支持两种传图方式，直接拖入工具，或复制文件后按`ctrl+v`粘贴（MacOS也需用ctrl+v，command+v无效），可根据图片修改代码，但还原精度有限（字体/间距易偏差）。
2. **MCP Server精准还原（以Figma为例）**
   - MCP作用：大模型与外部工具的沟通渠道，可获取设计稿的精准信息（间距、字体、样式等）；
   - 步骤：安装对应MCP Server→重启工具→`/resume`命令恢复历史会话（或启动时加`-c`自动恢复）→`/mcp`命令鉴权→粘贴设计稿链接，工具自动调用MCP工具（`get design context`/`get screenshot`）实现精准还原。

### （三）上下文管理
1. **压缩**：`-compact`命令，可追加压缩策略，保留关键信息，降低token消耗、提升工具性能，`ctrl+o`查看压缩结果；
2. **清除**：`-clear`命令，清空所有上下文，适用于新任务与历史内容无关的场景；
3. **项目记忆文件（CLAUDE.md）**
   - 生成：`-init`命令自动生成，支持中英文转换；
   - 特性：分**项目级别（当前目录，仅对本项目生效）**和**用户级别（用户目录，对当前用户所有项目生效）**，可手动修改添加项目要求/注意事项，工具启动时自动读取；
   - 快捷打开：`-memory`命令选择对应级别，直接打开编辑。

## 四、高级功能扩展与定制
### （一）Hook：自定义工具执行逻辑
1. 作用：在工具运行**前/后/失败/发送通知**等时机，执行用户指定的逻辑（如代码自动格式化）；
2. 配置步骤：`-hooks`命令进入配置页→选择执行时机（如`post tool use`工具使用后）→选择触发工具（如`write/edit`创建/编辑文件）→输入自定义命令→选择保存级别；
3. 保存级别：本地项目级（仅本机本项目，不共享）、项目级（所有项目用户共享）、用户级（仅当前用户生效）；
4. 实战示例：通过`jq`解析工具返回的文件路径，传递给`prettier`实现代码自动格式化。

### （二）Agent Skill：动态加载的Prompt模板
1. 作用：将固定格式/需求封装为模板，避免重复输入，工具可自动识别需求并调用，也支持用户主动调用；
2. 创建步骤：在用户目录`.their.cloud.skills`下创建文件夹→新建`skill.md`（含名称、描述、具体模板内容）→重启工具→`--skills`命令验证是否识别；
3. 调用方式：工具自动识别需求发起调用，或用户通过`--技能名称+具体请求`主动调用，结果更可控。

### （三）SubAgent：独立的子Agent
1. 特性：拥有**独立上下文、独立工具、独立Skill**，开辟全新对话窗口处理任务，仅返回最终结果，不污染主对话上下文；
2. 创建步骤：`create new agent`→选择级别（项目/用户）→选择创建方式（推荐工具初始化）→描述任务→选择可用工具/模型/展示颜色→编辑完善任务要求；
3. 实战场景：适合代码审核、大项目分析等对主上下文影响大的任务。

### （四）Agent Skill与SubAgent核心区别
| 特性         | Agent Skill                | SubAgent                    |
|--------------|----------------------------|-----------------------------|
| 上下文处理   | 继承并共享主对话上下文     | 拥有完全独立的上下文        |
| 中间过程     | 所有日志/思考记录到主上下文 | 中间过程仅在独立窗口，不回传 |
| Token消耗    | 易飙升，可能导致记忆过载    | 主对话无额外消耗，更高效    |
| 适用场景     | 与主上下文关联大、影响小的任务（如写开发日报） | 与主上下文关联小、影响大的任务（如代码审核） |

### （五）Plugin：一站式能力包
1. 作用：将Skill、SubAgent、Hook、MCP等能力打包，一键安装即可获得整套功能，类似软件安装包；
2. 操作命令：`-plugin`进入插件管理器，支持**discover（发现插件）、installed（已安装）、marketplaces（插件市场）**；
3. 安装与使用：选择插件后确定安装范围（当前用户/当前项目/用户+项目）→重启工具→工具自动识别并调用插件能力；
4. 生态：插件市场持续更新，涵盖前端设计、特定编程语言LSP等，也可自定义打包配置分享给团队/社区。

## 五、通用技巧与总结
1. **同类工具通性**：Claude Code的使用逻辑可迁移至Codex、Open Code等同类编程Agent，掌握后可一通百通；
2. **效率提升**：合理搭配自动模式、后台任务、Hook，减少手动操作；复杂任务先规划再执行，避免返工；
3. **风险规避**：谨慎使用`dangerously-skip-permissions`；重要项目用Git做精准版本控制，不依赖工具自身回滚；
4. **定制化核心**：通过CLAUDE.md统一项目要求，Hook标准化操作流程，Skill封装固定模板，SubAgent处理独立重任务，Plugin快速扩展能力，实现工具深度适配个人/团队开发需求。
