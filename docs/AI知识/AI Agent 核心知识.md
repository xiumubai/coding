# AI Agent核心知识笔记：概念、原理与两大构建模式（附简化版Claude Code实现）
本内容围绕**AI Agent**展开，从核心概念、本质价值讲起，重点解析目前最主流的**ReAct**和**Plan-And-Execute**两大构建模式，还通过实战演示了如何从零打造一个简化版Claude Code，彻底讲清AI Agent的底层运作逻辑，以下是核心知识点总结：

## 一、AI Agent 核心概念与本质
1. **定义**：将**大模型（大脑）**与**各类工具（感官/四肢）**组装而成的智能程序，能自主**感知外部环境**（如查询本地文件、网络信息）和**改变外部环境**（如写入代码、运行终端命令），实现任务的全自动化处理。
2. **大模型的局限性**：纯大模型（如GPT-4o、DeepSeek）无法感知/改变外界，比如能写代码但无法自行写入文件，需用户手动配合；需主动提供已有信息，否则无法自主查询。
3. **Agent的核心价值**：弥补大模型的局限性，通过工具让大模型拥有“动手能力”，无需用户干预即可独立完成复杂任务。
4. **常见类型与案例**
    - **编程类**：Claude Code、Cursor，可自主写代码、改代码、运行程序；
    - **综合类**：Minus，可自主生成执行计划、网络搜索、整理结果并生成报告；
    - 其他：做PPT、深度搜索、数据分析等领域的专属Agent。

## 二、ReAct 模式：目前使用最广泛的Agent构建模式
ReAct是**Reasoning and Acting（思考与行动）**的缩写，2022年10月由论文提出，是Agent最基础、最主流的运行模式，核心是**“思考→行动→观察”的循环**，直到完成任务。

### 1. ReAct 核心运行流程
用户提交任务后，Agent按以下步骤循环执行，直至输出最终答案：
1. **思考（Thought）**：Agent分析任务，判断是否需要调用工具、调用哪个工具；
2. **行动（Action）**：根据思考结果，请求调用对应的工具执行操作（如读写文件、运行终端）；
3. **观察（Observation）**：获取工具的执行结果（如“写入文件成功”“查询到XX信息”）；
4. **循环/结束**：根据观察结果再次思考，重复“行动-观察”；若无需再调用工具，直接输出**最终答案（Final Answer）**。

### 2. ReAct 实现核心：系统提示词（Prompt）
ReAct模式的运作并非依赖大模型的原生能力，而是通过**定制化系统提示词**约束大模型的输出逻辑，系统提示词会与用户任务一起传给大模型，相当于给大模型定了“迷你剧本”。
#### 系统提示词的核心组成（5部分）
- **职责描述**：明确大模型需按“思考-行动-观察”流程执行任务；
- **示例**：给出具体的任务执行案例，让大模型理解流程；
- **可用工具**：列出Agent支持的所有工具（如读文件、写文件、运行终端）；
- **注意事项**：规定执行过程中的规则；
- **环境信息**：提供当前操作系统、工作目录、文件列表等基础信息。

### 3. ReAct 实战：从零打造简化版Claude Code
以**实现一个能自动写贪吃蛇游戏的AI Agent**为例，核心是**“系统提示词+配套代码”**，代码核心为Python实现，整体流程和关键代码逻辑如下：
#### （1）核心准备
- 工具列表：封装3个核心工具函数（读文件、写文件、运行终端命令），工具是Agent的“四肢”；
- 模型选择：对接GPT-4o大模型，作为Agent的“大脑”；
- 工作目录：指定任务执行的文件夹（如snake文件夹），用于存放生成的代码。

#### （2）核心代码结构
- 主程序：`agent.py`，作为Agent的入口，负责接收用户任务、初始化参数；
- 核心类：`React_Agent`，封装Agent的所有核心逻辑，核心函数为`run()`；
- 系统提示词模板：包含占位符（工具列表、环境信息等），运行时自动渲染补全。

#### （3）`run()`函数核心逻辑（ReAct循环的核心）
1. 构建消息列表：包含**渲染后的系统提示词**和**用户任务**；
2. 调用大模型：获取返回结果，提取**Thought**部分并打印；
3. 结果判断：
   - 若包含`Final Answer`：直接返回，流程结束；
   - 若包含`Action`：解析出工具名和参数，执行对应工具函数；
4. 记录结果：将工具执行结果作为**Observation**加入消息列表；
5. 循环执行：重新调用大模型，传入更新后的消息列表，重复上述步骤。

#### （4）实战效果
Agent可自主完成**思考→写index.html→写css→写js**的全流程，无需用户干预，最终生成可直接运行的贪吃蛇游戏代码，实现与Claude Code类似的核心功能。

### 4. ReAct 完整运行时序（四大角色交互）
参与角色：**用户**、**Agent主程序**（串联流程）、**大模型**（思考）、**工具**（执行）
1. 用户提交任务→Agent主程序接收；
2. 主程序调用大模型→大模型返回Thought和Action；
3. 主程序打印信息，调用Action指定的工具→工具执行并返回结果；
4. 主程序将工具结果作为Observation加入历史消息，**再次调用大模型**；
5. 重复2-4步，直到大模型返回Thought和Final Answer；
6. 主程序打印最终结果，流程结束。

## 三、Plan-And-Execute 模式：先规划、再执行、可动态调整
这是一种**“先制定计划，再分步执行，根据执行结果动态修改计划”**的构建模式，Claude Code、Minus等主流Agent均采用此模式，相比ReAct更适合处理**复杂、多步骤**的任务，核心是“**规划→执行→复盘调整**”。

### 1. Plan-And-Execute 核心组成模块
该模式的Agent是**“Agent套Agent”**的设计，内部包含4个核心模块，模块可复用同一大模型，也可拆分使用不同模型：
1. **Plan模型**：负责根据用户任务，生成**具体、可执行的分步计划**；
2. **Reply模型**：负责根据**执行结果**，动态修改原计划，生成新计划；若任务完成，直接返回最终答案；
3. **执行Agent**：负责执行计划中的每一个步骤，可采用ReAct模式构建，内置对应工具（如网络搜索、文件操作）；
4. **Agent主程序**：负责串联所有模块，协调规划、执行、复盘的全流程。

### 2. Plan-And-Execute 核心运行流程
以“查询2025年澳网男子冠军的家乡”为例，流程如下：
1. **用户提需求**：将任务发给Agent主程序；
2. **生成初始计划**：主程序调用Plan模型，生成分步计划（如“查当前日期→查对应年份澳网男单冠军→查冠军家乡”）；
3. **分步执行+动态调整**：主程序将计划传给执行Agent，执行**第一步**，获取结果后，将**用户需求+原计划+执行记录**发给Reply模型，Reply模型生成**新计划**（删除已完成步骤，细化待执行步骤）；
4. **循环执行**：重复“执行单步→复盘调整计划”，直到完成所有步骤；
5. **输出最终答案**：最后一次复盘时，Reply模型判断任务完成，直接返回最终答案，主程序转发给用户，流程结束。

### 3. 核心特性
- **灵活性**：计划可根据执行结果动态修改，适配复杂任务的突发情况；
- **解耦性**：规划、执行、复盘模块相互独立，可单独优化（如更换执行Agent的运行模式）；
- **可解释性**：分步计划清晰，用户可明确看到Agent的执行步骤，便于追溯和调试。

## 四、ReAct 与 Plan-And-Execute 核心对比
| 特性         | ReAct 模式                | Plan-And-Execute 模式      |
|--------------|--------------------------|--------------------------|
| 核心逻辑     | 思考→行动→观察，循环执行   | 规划→执行→复盘，动态调整计划 |
| 计划性       | 无前置计划，走一步看一步   | 先制定完整计划，分步执行     |
| 复杂度       | 简单，适合单步骤/简单任务  | 复杂，适合多步骤/复杂任务    |
| 灵活性       | 低，仅能根据当前结果调整   | 高，可全局动态修改计划       |
| 可解释性     | 低，步骤零散，无整体规划   | 高，计划清晰，步骤可追溯     |
| 实现难度     | 低，核心是系统提示词+简单循环 | 高，需拆分模块，实现Agent嵌套 |
| 典型应用     | 简单文件操作、单步查询     | Claude Code、Minus、复杂项目开发 |

## 五、AI Agent 核心底层总结
1. **核心构成**：所有AI Agent的本质都是**“大模型+工具+调度逻辑”**，大模型负责思考，工具负责执行，调度逻辑（ReAct/Plan-And-Execute）负责串联二者；
2. **调度逻辑的重要性**：大模型和工具是基础，**调度逻辑决定了Agent的运行效率和适用场景**，是Agent构建的核心；
3. **工具的本质**：工具就是可被调用的**函数/程序**，只要能封装成可调用的接口，均可作为Agent的工具；
4. **系统提示词的价值**：对于基于大模型的Agent，系统提示词是**“规则制定者”**，决定了大模型的输出逻辑和Agent的运行流程。
