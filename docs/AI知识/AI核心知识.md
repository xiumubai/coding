# AI底层逻辑核心知识笔记：从LLM到核心概念全拆解
本内容从**工程视角**层层拆解AI领域的核心基础概念（LLM、Token、Context、Prompt等），讲清各概念的本质、关联和底层运作逻辑，帮你打通AI基础认知，以下是核心知识点总结：
## 一、最底层核心：LLM（大语言模型，Large Language Model）
1. **技术根基**：几乎所有大模型都基于**Transformer架构**开发，该架构由Google团队2017年在《Attention is All You Need》论文中提出，OpenAI将其落地并引爆AI浪潮。
2. **发展里程碑**
    - 2022年底：GPT-3.5发布，首个**可用级别**大模型，开启AI普及；
    - 2023年3月：GPT-4发布，大幅拉高AI能力天花板；
    - 现阶段：GPT-5.4成业界标杆，Claude、Gemini等后起之秀同台竞技，AI赛道不再是OpenAI独角戏。
3. **底层运行原理**：本质是**文字接龙游戏**，核心为“逐词预测生成”
    - 模型接收输入后，先预测**概率最高的下一个词**并输出；
    - 将输出的词追加到原输入后，再次预测下一个词，循环往复；
    - 直到模型判断表达完成，输出**特殊结束标识符**，生成过程终止。
4. **本质属性**：大模型是一个**庞大的数学函数**，内部仅做矩阵计算，**只认识数字，不识别文字**，需中间环节完成“文字-数字”的转换。

## 二、文字与数字的桥梁：Tokenizer（分词器）+ Token（令牌）
Tokenizer是人类与大模型之间的“翻译官”，Token是大模型处理文本的**最小基本单位**，二者配合实现文字和数字的双向转换。
### 1. Tokenizer的核心功能：编码+解码
#### （1）编码（文字→数字）：分两步完成，为大模型提供可处理的输入
1. **切分**：将人类输入的文字拆分成一个个最小文本片段，即**Token**；
2. **映射**：为每个Token分配唯一的数字标识（**Token ID**），形成“Token-Token ID”一对一绑定关系，最终将文字转换为**Token ID列表**传入大模型。
#### （2）解码（数字→文字）：仅一步映射，将大模型输出还原为人类可理解的文字
大模型输出单个Token ID，Tokenizer将其反向映射为对应Token，若模型未完成生成，则重复解码并追加结果，直至生成结束标识符。
### 2. Token的核心特性
1. **非字词对应**：Token与自然语言中的“字/词”无固定一对一关系，是模型自学习的**文本切分规则**，比如“工作坊”切分为“工作”+“坊”、“程序员”切分为“程序”+“员”，英文中“helpful”切分为“help”+“ful”。
2. **数量换算**：平均1个Token≈0.75个英文单词≈1.5-2个汉字；40万Token≈60-80万汉字/30万英文单词。
3. **处理规则**：大模型始终**逐Token接收输入、逐Token输出结果**，这也是大模型“一字一句生成答案”的根本原因。

## 三、大模型的“临时记忆”：Context（上下文）+ Context Window（上下文窗口）
大模型本身无实际记忆能力，其“记住聊天内容”的特性，由Context和Context Window共同实现。
1. **Context（上下文）**
    - 定义：大模型每次处理任务时**接收的信息总和**，是大模型的**临时记忆体**；
    - 包含内容：用户当前问题、历史对话记录、模型已输出的Token、工具列表、系统提示词（System Prompt）等；
    - 实现逻辑：每次向大模型发请求时，程序会自动将**历史对话+当前问题**一起传入，模型基于完整信息处理，从而实现“记忆”效果。
2. **Context Window（上下文窗口）**
    - 定义：Context能够容纳的**最大Token数量**，是大模型的“记忆容量上限”；
    - 主流规格：GPT-5.4（105万Token）、Gemini 3.1 Pro（100万Token）、Claude Opus 4.6（100万Token），100万Token约等于150万汉字，可容纳《哈利波特》全集内容；
    - 核心意义：决定大模型能处理的文本长度，窗口越大，可处理的信息越完整。
3. **大窗口解决方案**：当待处理文本（如上千页产品手册）超过Context Window上限时，可通过**RAG技术**优化
    - 原理：从海量文本中**抽取与用户问题最匹配的片段**，仅将这些片段与问题一起传入大模型；
    - 价值：突破窗口容量限制，同时大幅降低Token消耗和使用成本。

## 四、大模型的“指令入口”：Prompt（提示词）
1. **本质定义**：用户发给大模型的**具体问题或指令**，是大模型启动运行的唯一输入入口，比如“帮我写一首诗”就是一个基础Prompt。
2. **核心影响**：Prompt的质量直接决定大模型的输出效果——模糊的Prompt会导致输出结果不可控（如写诗可能是古诗/现代诗/打油诗）；清晰、具体的Prompt会让输出更贴合预期（如“写一首五言绝句，主题为秋天的落叶，风格悲凉”）。
3. **Prompt Engineering（提示词工程）**
    - 定义：研究如何设计高效Prompt，让大模型精准理解用户意图的领域；
    - 现状：如今逐渐淡化，原因一是**门槛低**，本质就是“把话说清楚”；二是**大模型能力提升**，即使Prompt含糊，也能大致猜出用户意图，无需额外花功夫设计。

## 五、核心概念底层关联总结
AI的基础运作逻辑是**“人类输入→Tokenizer编码→大模型处理→Tokenizer解码→人类输出”**，各核心概念的串联关系为：
1. 人类输入**Prompt（提示词）**，作为大模型的任务指令；
2. Tokenizer将Prompt**切分为Token**，并映射为**Token ID列表**（编码），传入**LLM（大模型）**；
3. 大模型结合**Context（上下文，含历史/当前信息）**，在**Context Window（上下文窗口）**上限内，逐Token预测生成结果，输出Token ID；
4. Tokenizer将Token ID反向映射为Token，拼接为人类可理解的文字（解码），完成输出。
